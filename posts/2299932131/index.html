第七章 容器和集合

容器

用来保存对象，并分为 Collection 和 Map。

Collection

源码分析： Collection
接口，集合类型，和数组类似，但是灵活性比数组高。

  • 结构图
    flowchart BT
Set --> Collection
TreeSet --> NavigableSet --> SortedSet --> Set
LinkedHashSet --> HashSet --> Set
ArrayList --> List --> Collection
LinkedList --> List
Vector --> List
BlockingQueue --> Queue
Deque --> Queue --> Collection

子类接口主要有：

  • List 按顺序保存元素
  • Set中的元素不能有重复元素
  • Queue 像排队一样，在尾部添加数据，在头部取出数据

List

源码分析： List
有序（存入的顺序和取出的顺序一致）、可重复、可以通过下标访问

  • ArrayList：底层是数组，查找快，增删慢
  • LinkedList：底层是链表，增删快，查找慢
  • Vector：底层是数组，线程安全

Set

源码分析： Set
无序、不可重复、不可以通过下标访问

  • HashSet：底层是单列 HashMap
  • TreeSet：由单列 TreeMap 实现，内部是红黑树结构，可对存入元素进行排序
  • LinkedHashSet：底层通过 LinkedHashMap 实现

Map

源码分析： Map
键值对(key-value)对象，可以通过键(key)来进行查找。key是无序且唯一。

  • 结构图
    flowchart BT
Hashtable --> Map
LinkedHashMap --> HashMap --> Map
IdentityHashMap --> Map
NavigableMap --> SorttedMap --> Map
WeakHashMap --> Map

HashMap

源码分析：HashMap
底层是由数组和链表组成，数组是主体，链表是为了解决 Hash 冲突，当链表长度大于8时，会转换为红黑树

HashTable

源码分析：HashTable
底层由数组和链表组成，线程安全

LinkedHashMap

源码分析：LinkedHashMap
继承自 HashMap，在 HashMap 的基础上维护了一个双向链表来记录元素添加的顺序，是的遍历的结果遵从插入的顺序，所以 LinkedHashMap 是有序的

TreeMap

源码分析：TreeMap
底层是红黑树，每个键值对是红黑树的一个节点，默认是安键进行升序排序，也可以指定排序的比较器，当用Iterator遍历 TreeMap 时，得到的数据是经过排序的。key不能为空，非同步。
