第八章 异常

异常

有些错误无法在编译期间找出来，所以需要在运行时来进行解决。这样就需要知道当出现异常时需要通过什么方式进行处理。

异常关键字

  • try: 异常执行代码
  • catch：捕获异常
  • finally：最终执行
  • throws：声明异常
  • throw：抛出异常

捕获异常

异常处理 try-catch

当出现异常时，如果需要处理的时候，就需要 try-catch 来将异常捕获

但是这种办法只能处理一个异常，那如果需要处理多个异常时，就需要多个 catch 来进行捕获，需要注意的是，如果有多个异常时需要将异常范围小的放在一场范围大的前面，如果异常范围大的放在异常范围小的前面时，那么异常范围小的永远都不会执行。

栈轨迹(堆栈信息)

在捕获的异常中，可以通过 e.printStackTrace() 来将异常所在位置的堆栈信息进行打印，也就是说可以通过这个将异常位置的调用链打印出来。

重新抛出异常

有时候代码抛出了 A 异常，但是我们需要改成 B 异常进行抛出，这时候可以在 catch 中重新将异常抛出。

finally 异常最终执行

在有些代码中，无论是否有异常都需要执行代码时，那么就需要 finally 。

注意：在 finally 中也可以抛出异常，当 在 finally 中抛出异常的时候，导致原来的异常被替换成新的异常

声明异常

有时候，其他代码调用我们写的代码的时候，会存在异常，但是这个异常我们希望不是我们进行处理，而是交给调用者来处理，这时候就可以通过 throws 来声明一个异常。

Error

在执行过程的遇到的硬件或操作系统导致的错误。这个错误对程序而言是致命的，会导致程序无法正常运行。例如 OutofMemoryError 因为程序无法处理这个异常，只能依靠外界进行干预，且 Error 错误属于系统内部错误，由 JVM 抛出，交给系统进行处理。

Exception

程序在运行中出现了可以预料的情况。比如 NullPointerException ，异常出现可以导致程序无法继续往下执行，但是这个异常可以进行捕获，使程序继续往后运行。

编译时异常

可检查异常，通常是语法错误和环境造成的异常。这种异常 特点是在方法后面有throws，这种异常必须捕获或继续往上抛出异常。

运行时异常

不检查异常 RuntimeException， 这些异常一般是由程序逻辑错误引起的，也就是语义错误。运行时异常因该在程序测试时暴露出来，然后进行去调试，这种异常一般尽量不要去捕获，而是去查找并修复，因为这种异常大部分情况都属于没有意料到的问题.

使用异常的技巧

  1. 异常不能代替判断，因为捕获异常是一件很耗时间和性能的操作
  2. 可以在用 一个try多个catch 进行捕获的时候就不要用多个 try-catch 进行捕获
  3. 用好异常的层次，尽量使用和捕获具体的异常而不是父类异常，如：RuntimeException
  4. 将异常打印出来，而不是忽略
  5. 在捕获异常时，要捕获具体很细的异常，而不是简单的一个 Exception
  6. 当知道有一场时，可以将异常向上传递出去
